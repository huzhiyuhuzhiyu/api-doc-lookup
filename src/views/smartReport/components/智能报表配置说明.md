# CommonReport 组件使用说明

## 概述

CommonReport 是一个通用报表组件，提供了丰富的功能来展示和操作表格数据。它支持动态表头、数据汇总、导出功能、高级查询等特性，可以大大简化报表开发工作。

## 功能特性

- **动态表头**: 支持根据数据动态生成表头
- **数据汇总**: 支持在表格底部显示合计行
- **数据导出**: 支持将表格数据导出为 Excel 文件
- **高级查询**: 支持复杂的查询条件配置
- **自定义列**: 支持自定义列的显示和格式化
- **分页功能**: 内置分页组件，支持数据分页展示

## 使用方法

### 基本用法

```vue
<template>
  <CommonReport :config="config"/>
</template>

<script>
import CommonReport from '@/views/smartReport/components/CommonReport.vue'

export default {
  components: { CommonReport },
  data() {
    return {
      config: {
        // 配置项
      }
    }
  }
}
</script>
```

### 配置项说明

#### 主要配置项

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fetchApi | Function | 是 | 获取数据的 API 接口函数 |
| initQuery | Object | 否 | 初始化查询参数 |
| dynamicHeaders | Object | 否 | 动态表头配置 |
| columnsConfig | Array | 否 | 静态列配置 |
| summaryConfig | Object | 否 | 汇总行配置 |
| exportConfig | Object | 否 | 导出功能配置 |
| superQueryConfig | Object | 否 | 高级查询配置 |
| rowKey | String | 否 | 行数据的唯一标识字段，默认为 'id' |

#### 动态表头配置 (dynamicHeaders)

```javascript
dynamicHeaders: {
  enable: true, // 是否启用动态表头
  config: {
    // 左侧固定列配置
    leftFields: [
      {
        prop: 'workshopName',
        label: '车间名称',
        sortable: 'custom'
      }
    ],
    
    // 动态列配置
    dynamicColumns: [
      {
        type: 'chinese', // chinese模式：中文关键词匹配 exact模式：精确字段名匹配
        minWidth: 120,
        children: [ // 可选子列配置 children为空则匹配 " 测试A：0 " 类型
          { prop: 'qualifiedQuantity', label: '合格数' },
          { prop: 'unqualifiedQuantity', label: '不良数' }
        ]
      }
    ],
    
    // 日期列特殊配置
    dateColumns: {
      mergeParent: true, // 是否合并日期父列
      children: [ // 子列配置
        {prop: 'qualifiedQuantity', label: '合格数', minWidth: 100},
        {prop: 'unqualifiedQuantity', label: '不合格数', minWidth: 100}
      ],
      datePattern: /^\d{4}-\d{2}-\d{2}$/, // 日期匹配正则表达式
      dateFormatter: date => date.replace(/-/g, '/') // 可选的日期格式化函数
    }
  }
}
```

#### 汇总行配置 (summaryConfig)

```javascript
summaryConfig: {
  showSummary: true, // 控制是否显示汇总行
  summaryMapping: {
    // 字段映射，用于指定显示字段与实际数据字段的对应关系
    'qualified': 'qualifiedQuantity',
    'defect': 'unqualifiedQuantity'
  }
}
```

#### 导出配置 (exportConfig)

```javascript
exportConfig: {
  enable: true, // 是否启用导出功能
  exportType: 'PROD_REPORT', // 导出类型
  exportName: '生产报表_' + moment().format('YYYYMMDD'), // 导出文件名
  exportHidden: true, // 是否导出隐藏列
  fetchApi: null // 自定义导出API接口
}
```

#### 高级查询配置 (superQueryConfig)

```javascript
superQueryConfig: {
  systemSearchView: [], // 系统预设查询视图
  superQueryJson: [] // 高级查询条件
}
```

#### 列配置 (columnsConfig)

当不使用动态表头时，可以通过 columnsConfig 配置静态列：

```javascript
columnsConfig: [
  { 
    prop: "partnerCode", 
    label: '供应商编码', 
    align: 'left', 
    minWidth: 180,
    sortable: true, // 是否可排序
    fixed: false, // 是否固定列
    formatter: (row, column, cellValue) => { // 格式化函数
      return cellValue;
    }
  }
]
```

### 完整示例

```vue
<template>
  <CommonReport :config="config"/>
</template>

<script>
import CommonReport from '@/views/smartReport/components/CommonReport.vue'
import { getProductionData } from '@/api/report'

export default {
  name: "ProductionReport",
  components: { CommonReport },
  data() {
    return {
      config: {
        // 数据获取配置
        fetchApi: getProductionData,
        
        // 初始化查询参数
        initQuery: {
          reporting: '',
          input: '',
          pageNum: 1,
          pageSize: 20,
        },
        
        // 动态表头配置
        dynamicHeaders: {
          enable: true,
          config: {
            leftFields: [
              {
                prop: 'workshopName',
                label: '车间名称',
                sortable: 'custom'
              }
            ],
            dynamicColumns: [
              {
                type: 'chinese',
                minWidth: 120,
                children: [
                  { prop: 'qualifiedQuantity', label: '合格数' },
                  { prop: 'unqualifiedQuantity', label: '不良数' }
                ]
              }
            ]
          }
        },
        
        // 汇总行配置
        summaryConfig: {
          showSummary: true,
          summaryMapping: {
            '合格数': 'qualifiedQuantity',
            '不良数': 'unqualifiedQuantity'
          }
        },
        
        // 导出配置
        exportConfig: {
          enable: true,
          exportType: 'PROD_REPORT',
          exportName: '生产报表_' + moment().format('YYYYMMDD'),
          exportHidden: true
        },
        
        // 高级查询配置
        superQueryConfig: {
          systemSearchView: [{
            matchLogic: "AND",
            fullName: "默认视图",
            conditionJson: {
              condition: [
                {
                  prop: 'workshopName',
                  symbol: 'like',
                  fixed: true
                },
                {
                  prop: 'createTime', // 属性*
                  // value: [this.jnpf.getToday('YYYY-MM-DD HH:mm:ss', 'today-29'), this.jnpf.getToday('YYYY-MM-DD HH:mm:ss', 'todayLastMoment')], // 默认值
                  symbol: 'between', // 比较符*
                  timeOffset: true, // 保存视图后的静态时间区间随实际查询时刻偏移
                  fixed: true // 是否在搜索栏显示
                }
              ],
              keywordQuery: this.jnpf.getKeywordQuery('product'), // 带有产品信息的表使用此预设
              pageSize: 20,
              orderItems: [{
                asc: false,
                column: ""
              }]
            }
          }],
          superQueryJson: []
        },
        
        rowKey: 'id'
      }
    }
  }
}
</script>
```

## 注意事项

1. **fetchApi 必须提供**: 这是组件获取数据的核心配置项，必须提供一个返回 Promise 的函数。
2. **动态表头与静态列**: 当启用动态表头时，静态列配置 (columnsConfig) 将被忽略。
3. **数据结构要求**: 使用动态表头时，返回的数据需要符合特定的结构要求。
4. **性能考虑**: 对于大量数据的场景，建议合理配置分页大小以保证性能。
